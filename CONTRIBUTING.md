By participating in this project, you are expected to uphold our Code of Conduct.

üêõ Reporting Bugs

If you find a bug (e.g., the FPS counter isn't working on a specific browser, or a styling issue appears), please follow these steps:

Search existing Issues: Check the repository's Issues tab to see if the bug has already been reported.

Create a New Issue: If not reported, create a new issue with a clear title.

Details to Include:

A concise description of the bug.

Steps to reproduce the behavior.

The expected behavior.

Your environment (Browser, OS, device type).

üí° Suggesting Enhancements

We welcome suggestions for new features, design improvements, or code refactoring. Please open an issue and use the enhancement label to describe your idea.

üíª Making a Contribution (Pull Requests)

Fork the Repository: Fork this repository to your own GitHub account.

Create a Branch: Create a new branch for your feature or fix (e.g., git checkout -b feature/add-dark-mode or fix/typo-in-readme).

Commit Changes: Commit your changes with a descriptive commit message.

Open a Pull Request (PR): Push your branch to your fork and open a Pull Request against the main branch of this repository.

Please fill out the provided Pull Request Template completely when submitting your changes.
